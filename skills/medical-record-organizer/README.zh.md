# 🏥 病历归档整理（medical-record-organizer）

> 自动整理、分类、归档中文病历文件，为癌症及慢性病患者建立结构化的个人病历库。

## 适用人群

- 肿瘤患者或家属，积累了大量检查单、CT 报告、病理报告需要整理
- 慢性病患者（糖尿病、高血压、慢性肾病等），需要系统管理多年病历
- 代替家人管理病历的照护者

## 功能特性

- **自动分类归档**：识别并分入 10 类目录（影像、化验、病理、基因检测、出院小结等）
- **批量导入**：支持 zip / rar / 7z 压缩包一次性导入多份文件
- **支持文件类型**：PDF 报告、手机拍照的纸质报告（图片）、文本文件
- **提取关键指标**：自动从化验单中识别 CEA、CA-199、AFP、CA-125 等肿瘤标志物数值
- **识别分子病理**：NGS 基因检测（KRAS/EGFR/BRCA/TMB/MSI）和免疫组化（PD-L1/HER2/Ki-67）
- **生成 INDEX.md**：每个病人一份导航索引，罗列所有文件及一行摘要，方便 Agent 快速定位
- **累计追踪肿标趋势**：每次归档自动更新 `肿标趋势.md`，无需手动录入
- **支持中英文病历**
- **隐私模式默认开启**：自动使用匿名患者ID建档，生成目录与索引默认禁止出现真实姓名、身份证号、手机号、详细地址等隐私信息

## 触发方式

直接告诉 Agent 你想存什么文件：

```
"帮我存这份CT报告"
"整理这份检查单"
"归档这个化验单"
"添加这份出院小结"
"把这个压缩包里的报告全部导入"
```

## 工作流程

1. **建档**：首次使用时，Agent 以匿名患者ID（如 `PT-1A2B3C4D5E`）创建专属目录（若输入姓名，自动转换并不落盘）
2. **识别输入**：判断是单个文件还是压缩包（批量模式）
3. **对照分类表**：根据内容关键词确定归入哪个类目，优先级：出院小结 > 完整病理 > 基因检测 > 影像 > 化验；无法判断的进入 `未分类/` 待确认
4. **提取信息**：读取日期、医院、科室、核心结论、异常指标
5. **写入归档文件**：以 `YYYY-MM-DD_文档类型_简述.md` 命名保存
6. **更新索引**：将新条目追加到 `INDEX.md` 和 `timeline.md`

## 生成的目录结构

```
~/.openclaw/patients/PT-1A2B3C4D5E/
├── INDEX.md                      ← 导航入口（Agent 读这里定位文件，无需逐一打开）
├── profile.json                  ← 匿名资料卡（禁止写入直接身份信息）
├── timeline.md                   ← 按时间排列的事件总表
├── 01_当前状态/
├── 02_诊断与分期/
├── 03_分子病理/                   ← 基因检测 + 免疫组化（单独一类，不与病理混放）
├── 04_影像学/                     ← CT / MRI / PET-CT / 超声
├── 05_检验检查/                   ← 血常规 / 生化 / 肿瘤标志物
│   └── 肿瘤标志物/肿标趋势.md      ← 累计追加，每次归档自动更新
├── 06_治疗决策历史/
├── 07_合并症与用药/
├── 08_出院小结/
├── 09_Apple_Health/              ← 苹果健康数据（由 apple-health-digest 写入）
└── 10_原始文件/
```

## 系统要求

| 依赖 | 说明 |
|------|------|
| Python 3 | 必须 |
| pdfplumber | PDF 文字提取（由 OpenClaw 自动安装） |
| rarfile | RAR 包支持（由 OpenClaw 自动安装） |
| unrar 或 unar | 系统级 RAR 解压工具（需手动安装，有其中一个即可） |

## 与其他 Skill 的关系

本 skill 是三个 skill 的**基础层**，建立的病人目录是其他两个 skill 的数据来源：

- **health-timeline**：读取本 skill 建立的 `timeline.md`、`INDEX.md` 和 `肿标趋势.md`，进行查询和可视化
- **apple-health-digest**：将苹果健康数据写入 `09_Apple_Health/`，并与治疗记录关联分析
