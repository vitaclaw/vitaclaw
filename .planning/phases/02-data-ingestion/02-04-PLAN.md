    ```python
    class OCRPipeline:
        """Orchestrate OCR extraction for Chinese medical documents."""

        DOC_TYPES = {
            "体检报告": "physical_exam",    # Table-heavy
            "检验单": "lab_test",           # Table-heavy
            "门诊病历": "outpatient",       # Text-heavy
            "处方单": "prescription",       # Text-heavy
        }

        TABLE_TYPES = {"physical_exam", "lab_test"}
        TEXT_TYPES = {"outpatient", "prescription"}

        def __init__(self, concept_resolver=None):
            self._concept_resolver = concept_resolver or ConceptResolver()
            self._table_extractor = None  # Lazy init
            self._text_extractor = None   # Lazy init

        def process(self, file_path: str, person_id: str | None = None) -> dict:
            """Process a medical document image/PDF.
            Returns staging dict (see interfaces section for format).
            DOES NOT write to HealthDataStore -- staging only per D-11."""

        def _prepare_input(self, file_path: str) -> list[str]:
            """Handle input: HEIC -> JPG, PDF -> images, return list of image paths.
            Per D-17: accepts JPG, PNG, HEIC, PDF."""

        def _classify_document(self, image_path: str) -> str:
            """Auto-detect document type from layout and keywords.
            Per D-12: system auto-detects, does not require user to specify.
            Returns one of DOC_TYPES values."""

        def _map_to_concepts(self, extracted_fields: list[dict]) -> list[dict]:
            """Map extracted item names to health concepts via ConceptResolver.
            Adds concept_id and record_type to each field."""

        def _archive_original(self, file_path: str, person_id: str | None = None) -> str:
            """Copy original document to memory/health/files/ with metadata.
            Per OCR-05: archive original for reference.
            Returns archived path."""
    ```

    Key implementation details:
    - `_prepare_input`: Reuse `_ensure_processable` pattern from redact_ocr.py for HEIC. Reuse `pdf_to_images` pattern from privacy_desensitize.py for PDF. Per D-17.
    - `_classify_document`: Use a simple heuristic -- run quick text OCR on first image, look for keywords:
      - "体检" or "健康体检" -> physical_exam
      - "检验" or "化验" or "生化" -> lab_test
      - "门诊" or "病历" -> outpatient
      - "处方" or "Rx" -> prescription
      - Default: lab_test (most common per D-12)
    - Dispatch: TABLE_TYPES -> TableExtractor, TEXT_TYPES -> TextExtractor
    - `_map_to_concepts`: Use ConceptResolver to map Chinese lab item names to concept IDs. For common items, maintain a lookup dict of Chinese -> concept_id mappings (e.g., "血红蛋白" -> "hemoglobin", "血糖" -> "blood-glucose", "血压" -> "blood-pressure")
    - `_archive_original`: Copy file to `memory/health/files/YYYY-MM-DD_{doc_type}_{hash}.{ext}`, write a metadata JSON alongside it
    - Output format: staging dict with document_type, source_file, archived_path, confidence, extracted_fields, raw_text
    - CRITICAL: This pipeline NEVER calls HealthDataStore.append(). It returns staging data only. Storage happens in Plan 05 after user confirmation.
