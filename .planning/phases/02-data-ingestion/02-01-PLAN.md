    Create `skills/health-onboarding/SKILL.md` with YAML frontmatter and a structured prompt body. Per D-01: this is a pure SKILL.md conversation, no Python UI.

    YAML frontmatter must include:
    - name: health-onboarding
    - description: Guided health profile setup through conversational flow
    - version: 1.0.0
    - user-invocable: true
    - allowed-tools: [Edit, Read, Write]
    - tags: [onboarding, profile, setup]

    Prompt body must instruct the AI to:

    1. **Read existing state first** (per D-04 merge requirement):
       - Read USER.md, IDENTITY.md, _health-profile.md
       - Identify which fields are "pending" or empty
       - If re-running, show current values and ask what to update

    2. **Conversational collection** (per D-02):
       - Basic demographics: age/date of birth, sex, height, weight baseline
       - Chronic conditions with diagnosis dates
       - Current medications with dosages and timing
       - Supplements with dosages
       - Primary health goals (e.g., "control blood pressure", "lose 5kg")
       - Preferred reminder cadence (daily, weekly, custom)
       - Care team info (primary doctor name, specialty, clinic)
       - The conversation must feel natural, not form-like. Group related questions, acknowledge answers, ask follow-ups.

    3. **Output writing** (per D-03):
       - USER.md: Update preferences section (reminder cadence, units, language)
       - IDENTITY.md: Update role and service target based on conditions/goals
       - _health-profile.md: Write all health data in structured markdown with sections: Baseline, Conditions, Medications, Supplements, Allergies, Risk Thresholds, Care Plan
       - Use Edit tool for targeted section updates, NOT Write for full replacement (per D-04 and Pitfall 5)

    4. **Family registration** (per D-03):
       - If person_id is not "self", instruct the AI to register the person in _family.yaml using FamilyManager
       - Include the Python snippet: `from skills._shared.family_manager import FamilyManager; fm = FamilyManager(); fm.add_member(person_id, display_name=name)`

    5. **Completion summary**:
       - After writing, display a summary of what was populated
       - List any fields left as "pending" for future updates
  </action>
