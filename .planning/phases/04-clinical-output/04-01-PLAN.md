    4. Recent lab results: Use `CrossSkillReader.read()` with concept "lab-result" or similar. Check what concepts are available.

    **_charts_as_base64(days):** For each chart path from _generate_charts, read bytes and base64-encode as `data:image/png;base64,...` per D-04. Use DPI=100 for embedded charts (per Pitfall 4 recommendation).

    **generate(days=30, format="markdown", write=True):** Main entry point.
    - Collect data via _collect_data(days)
    - Generate charts via _generate_charts(days)
    - Route to _render_markdown, _render_html, or _render_pdf per format
    - If write=True, save output to `memory/health/files/visit-summary-{date}.{ext}`
    - Return dict: {"path": str, "content": str, "format": str}

    **_render_html(data, charts):** Use Jinja2 with INLINE template string (per Pitfall 6 and research recommendation). Template stored as module-level `_VISIT_SUMMARY_HTML` constant. Self-contained HTML with inline CSS per D-03, D-09. Styling per D-10: light background (#fafafa), PingFang SC font stack, card layout for metrics, alternating table rows, mobile-responsive, print-friendly CSS. Embed charts as base64 PNG via _charts_as_base64 per D-04.

    **_render_pdf(data, charts):** Generate HTML first, then try `from weasyprint import HTML; HTML(string=html).write_pdf(path)`. On ImportError: save HTML and return fallback message per D-03, Pitfall 3.

    **Graceful degradation per Pitfall 7 and Pattern 3:** Every section checks for empty data. Show "近期无血压记录" / "近期无血糖记录" / "未找到用药记录" etc. instead of empty sections. Never crash on missing data.
