    2. Create skills/medical-document-ocr/scripts/ocr_llm_enhancer.py (per D-05):
       - Shebang, docstring, `from __future__ import annotations`
       - sys.path setup to import ConceptResolver from _shared
       - class OCRLLMEnhancer:
         - __init__(self, concept_resolver=None, llm_callback=None):
           - concept_resolver: ConceptResolver instance (or create default)
           - llm_callback: callable that takes a prompt string and returns LLM response string, or None for no-LLM mode
         - enhance(self, extracted_fields: list[dict], raw_text: str = "") -> list[dict]:
           - If llm_callback is not None: build prompt, call LLM, parse response to get corrected fields
           - If llm_callback is None or LLM fails: use original fields (graceful degradation per D-09)
           - For each field (enhanced or original):
             a. Look up concept_id via ConceptResolver if not already set
             b. Add loinc_code: call resolver.get_loinc(concept_id) (per D-07/AOCR-02)
             c. Add suggested_skill: call resolver.get_producers(concept_id)[0]["skill"] if available, else "" (per D-08/AOCR-03)
             d. If concept_id is empty/unknown: log "[WARN] Unmapped OCR field: {item_name}" to stderr (per D-07)
           - Return enhanced field list

         - _build_llm_prompt(self, extracted_fields: list[dict], raw_text: str) -> str:
           - Few-shot prompt for Chinese medical document field extraction
           - Include raw_text for context
           - Include current extracted_fields as "initial extraction" for LLM to correct
           - Request JSON output: list of {item_name, value, unit, reference_range, confidence}
           - Instruct LLM to: (1) correct misread values using medical context, (2) identify field types (per D-06), (3) add missing units

         - _parse_llm_response(self, response: str, original_fields: list[dict]) -> list[dict]:
           - Try json.loads on response (handle markdown code fences)
           - If valid JSON list of dicts: merge with original fields (keep concept_id from originals, update value/unit/confidence from LLM)
           - If parsing fails: return original_fields unchanged

    3. The llm_callback design (per D-05): The enhancer does NOT call any LLM API directly. It accepts a callback function that the host AI runtime provides. In tests, we mock this callback. In production, the OCR pipeline passes a callback that delegates to the host AI. This keeps the module decoupled from any specific LLM provider.
