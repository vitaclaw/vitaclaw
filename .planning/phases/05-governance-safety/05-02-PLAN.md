   - Markdown table columns: Skill | Grade | Score | Frontmatter | Code | Tests | Lint | Docs | Issues
   - Sorted worst-first (lowest score first)

Create `scripts/skill_staleness_scanner.py` per D-08, D-09:

1. Add function `extract_keywords(text: str) -> set[str]`:
   - Extract meaningful terms from text (lowercase, split on whitespace/punctuation)
   - Filter out common stop words (English and Chinese)
   - Keep terms >= 3 chars that look like identifiers (tool names, data types, format names)
   - Per Claude's Discretion: use simple word overlap (not TF-IDF) -- fast, deterministic, no dependencies

2. Add function `check_staleness(skill_dir: Path) -> dict | None`:
   - Read SKILL.md description from frontmatter
   - Read all `.py` files in the skill directory, concatenate their content
   - If no Python files exist, return None (prompt-only skill, not applicable)
   - Extract keywords from description and from Python code
   - Compute keyword match ratio: `len(description_keywords & code_keywords) / len(description_keywords)`
   - Per D-08: flag skills with <50% keyword match as potentially stale
   - Return: `{"slug": str, "match_ratio": float, "description_keywords": list, "matched": list, "unmatched": list, "suggestion": str}`
   - Per D-09: suggestions are one of: "Update SKILL.md description to match implementation", "Review Python implementation -- may be incomplete", "Mark as prompt-only skill if no Python is expected"

3. CLI with argparse:
   - `--format markdown|json` (default markdown)
   - `--threshold 0.5` (configurable match threshold)
   - `--output reports/skill-staleness-report.md`
   - Markdown output: table of flagged skills with columns: Skill | Match% | Unmatched Keywords | Suggestion

4. Add `#!/usr/bin/env python3` shebang and module docstring per project conventions.
