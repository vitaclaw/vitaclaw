# OpenClaw 大健康领域 57 个原子级 Skills 全景报告

> 生成日期：2026-03-05
> 状态：规划阶段

---

## 概述

基于 OpenClaw 框架，面向 5 类目标人群的大健康 Skills 体系。覆盖医疗病历管理、肿瘤专项、糖尿病管理、慢病管理、个人健康日常、健身减肥塑形增肌、生物医药技术、心理压力管理、家庭健康管理、可穿戴设备、健康数据聚合共 11 大领域，57 个不可再拆分的原子级 Skill。

### 设计原则

1. **原子性**：每个 Skill 只做一件事，输入明确、输出明确，不可再拆
2. **人群导向**：5 类目标人群（肿瘤患者、糖尿病患者、慢病患者、健身人群、日常健康管理者），每类有专属 Skill 组合
3. **隐私优先**：健康数据全部本地存储，不上传第三方
4. **复用优先**：优先复用 ClawHub 已有 Skill 和 MCP Server

### 目标人群（5 类）

| 人群 | 典型场景 | 对应分类 |
|------|---------|---------|
| **肿瘤患者/家属** | 病历管理、治疗追踪、临床试验匹配、副作用管理 | 一、二、七、九 |
| **糖尿病患者** | 血糖追踪、胰岛素管理、饮食控制、HbA1c 趋势 | 三 |
| **慢性病患者**（高血压/肾病/慢性疼痛等） | 指标监测、复查提醒、用药管理 | 四 |
| **健身/减肥/塑形/增肌人群** | 热量管理、蛋白质计算、身体成分、训练计划 | 六 |
| **日常健康管理者**（含技术人群） | 睡眠优化、补剂管理、体检解读、咖啡因追踪 | 五、八、九、十、十一 |

[包含具体步骤、分支逻辑、输出格式]

## 验证方式

1. 每个 skill 创建后，检查 SKILL.md YAML frontmatter 是否合规
2. 有脚本的 skill，验证脚本可独立运行（`py -3 scripts/xxx.py --help`）
3. 按人群场景端到端测试：
   - **肿瘤患者**：导入病历 → 生成时间线 → 肿瘤病程总结 → 匹配临床试验
   - **糖尿病**：记录血糖 → 计算胰岛素 → 饮食管理 → HbA1c 趋势
   - **健身**：设热量目标 → 记录饮食/训练 → 蛋白质追踪 → 体重趋势 → 周报
4. API 类 skill 验证：确认 API endpoint 可达、返回格式正确
